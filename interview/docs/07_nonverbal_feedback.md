# [7] 비언어 피드백

## 용도

MediaPipe.js 의 `FaceLandmarker` 가 클라이언트에서 측정한 비언어 지표를 받아 사용자에게 친절한 한국어 코칭을 생성. 점수는 20점 만점이며 최종 점수 100점 중 비언어 영역에 들어감.

## 호출 위치

- 파일: `nonverbal_feedback.py`
- 함수: `generate_nonverbal_feedback(metrics, *, model="gpt-4o-mini", api_key=None)`
- 모델: 항상 `gpt-4o-mini` 고정 (`NONVERBAL_FEEDBACK_MODEL` in `main.py`)

## 점수 구조 — 3축 통합 체계 (20점 만점)

| 항목 | 만점 | 만점 기준 |
|---|---|---|
| **표정 안정성** | 6점 | 미소 비율 ≥ 20% |
| **시선 안정성** | 10점 | 정면 응시 비율 ≥ 75% (응시 9점) + 분당 깜빡임 10~25회 (1점) **통합** |
| **자세 안정성** | 4점 | 프레임당 평균 이동 ≤ 1.5px |

이전 4축 체계(표정 6 + 시선 6 + 깜빡임 4 + 자세 4) 에서 **시선 + 깜빡임을 "시선 안정성 10점" 으로 통합**.
응시 만점 기준은 90% → **75%** 로 완화. 응시 판정 임계값(off-center)도 0.30 → **0.45** 로 완화.

**깜빡임 비중 축소** (2026-05): 사용자에게 깜빡임 코칭이 부담이 된다는 피드백을 반영,
점수 가중치를 4점 → **1점** 으로 축소, UI 차트·LLM 코칭에서도 깜빡임을 노출하지 않음.
내부적으로는 여전히 측정·기록되지만 (`metrics.blink_per_minute`) 사용자에게는 정면 응시만 보임.

내부적으로는 응시·깜빡임 컴포넌트가 분리되어 계산되며, 외부 노출용 `scores_20` JSON 에는 3축으로 표시:
- `scores_20.smile` (max 6)
- `scores_20.gaze`  (max 10, focus+blink 합산)
- `scores_20.posture` (max 4)
- `scores_20._focus_component`, `scores_20._blink_component` — 디버깅용 (UI 노출 X)

카메라 OFF 면 비언어 점수 0, 최종 점수는 80점 만점으로 표시.

## UI 노출 정책

- 면접 진행 중: **카메라 영상만** 보임. 실시간 인디케이터·랜드마크 포인트 클라우드 등 분석 시각화는 모두 제거.
- 결과/상세/분석 페이지: 3축 코칭 카드 (표정 안정성 · 시선 안정성 · 자세 안정성)

## SYSTEM_PROMPT

```
너는 비대면 면접 코치이다. 지원자의 면접 중 비언어 행동 측정 결과를 보고
구체적이고 친절한 한국어 조언을 작성한다.

평가 기준 (총 20점, 3축 통합):
  1. 표정 안정성  6점 — 미소 비율 20% 이상이면 만점
  2. 시선 안정성 10점 — 응시 비율 75% 이상(6점) + 분당 깜빡임 10~25회(4점) 통합
  3. 자세 안정성  4점 — 프레임당 평균 이동 1.5px 이하면 안정

가이드:
- 3가지 항목 각각에 대해 짧고 구체적인 코칭 한 줄을 작성한다.
- 좋은 점은 짧게 인정하고, 부족한 점은 실행 가능한 개선 행동으로 제시한다.
- "표정이 부족합니다" 식의 평가만 하지 말고 "거울 보며 자기소개 3회 연습" 같은 행동 단계를 제시한다.
- 시선 안정성 항목은 응시와 깜빡임을 종합해 한 줄 코칭을 작성한다.
  예: 응시는 양호하나 깜빡임이 많다 → "응시는 안정적이나 깜빡임이 다소 잦으니, 들숨에서 한 박자 시선을 머무는 연습."
- 종합 코멘트는 2~3문장으로 전체 인상을 정리한다.
- 인성/외모/성격은 절대 평가하지 않는다 (오직 측정 가능한 행동만).

출력 JSON 형식 (반드시 이 키만):
{
  "summary": "종합 2~3문장",
  "smile":   "표정 안정성 코칭 한 줄",
  "gaze":    "시선 안정성 코칭 한 줄 (응시 + 깜빡임 통합)",
  "posture": "자세 안정성 코칭 한 줄",
  "next_action": "다음 면접 전에 가장 우선 연습할 한 가지"
}
```

`metrics` 가 다음 형태로 전달됨 (JSON 인코딩):

```json
{
  "총점": 14.5,
  "지표": {
    "미소 비율 (%)":      12.4,
    "정면 응시 비율 (%)": 87.1,
    "분당 깜빡임":        18,
    "평균 자세 이동 (px/f)": 0.9
  },
  "항목별 점수 (3축)": {
    "표정 안정성":  {"점수": 3, "만점": 6, "라벨": "약간 부족"},
    "시선 안정성":  {
      "점수": 9, "만점": 10, "라벨": "시선 안정",
      "분해": {"응시(75% 만점)": 6, "깜빡임(10~25/min 만점)": 4}
    },
    "자세 안정성":  {"점수": 4, "만점": 4, "라벨": "안정"}
  },
  "진행 시간(초)": 35.2
}
```

## 안전 가드 + 폴백

- `metrics.ok=False` 면 LLM 호출 없이 안내 메시지 반환 ("측정 시간이 부족...")
- `metrics.scores_20.gaze` 가 없는 옛 세션 데이터는 `focus`+`blink` 컴포넌트를 합산해 폴백
- 인성/외모/성격 평가 금지 — system prompt 에서 강조
